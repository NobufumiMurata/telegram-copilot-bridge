# Telegram MCP テストプロンプト

以下のプロンプトを順番に Copilot Chat に入力して、各ツールの動作を確認してください。

---

## Test 1: 通知送信 (telegram_notify)

```
Telegram で以下のメッセージを送信して:
「✅ MCP テスト通知: 接続確認 OK」
```

**期待結果**: スマホに HTML メッセージが届く

---

## Test 2: 承認フロー (telegram_ask_approval)

```
Telegram で承認を求めて。質問は「テスト用 VM を削除してよいですか？」で、選択肢は「承認」「却下」「保留」の 3 つ。
```

**期待結果**: インラインボタン 3 つが届く → タップすると Copilot に結果が返る

---

## Test 3: 承認タイムアウト確認

```
Telegram で承認を求めて。質問は「タイムアウトテスト」で、タイムアウトは 1 分に設定して。ボタンは押さないで待って。
```

**期待結果**: 1 分後に「TIMEOUT」が返る（ボタンを押さずに待つ）

---

## Test 4: フリーテキスト応答 (telegram_wait_response)

```
Telegram で「次に何をしますか？」と聞いて、返事を待って。タイムアウトは 2 分で。
```

**期待結果**: スマホにプロンプトが届く → 自由テキストで返信 → Copilot にテキストが返る

---

## Test 5: ファイル送信 (telegram_send_file)

```
pyproject.toml を Telegram に送信して。キャプションは「プロジェクト設定ファイル」で。
```

**期待結果**: スマホにファイルが届く

---

## Test 6: 一連のフローテスト（統合）

```
以下の手順を実行して:
1. Telegram で「タスク開始します」と通知
2. Telegram で「本番環境にデプロイしてよいですか？」と承認を求めて
3. 承認されたら「デプロイ完了しました」と通知
4. 却下されたら「デプロイをキャンセルしました」と通知
5. 最後に Telegram で「次の指示をお願いします」と聞いて、返事を待って
```

**期待結果**: 通知 → 承認ボタン → 結果に応じた通知 → 次の指示待ち、の一連のフローが動く
